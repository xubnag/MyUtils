## 一、阶段一：说明
> 基于接口而非实现编程 实践
```TEXT
1.函数的名称不能暴露任何实现的细节
2.封装具体的实现细节
3.为实现类定义抽象的接口（暴露通用的功能给调用者，特殊的保留）例如：上传、下载、创建bucket作为通用的功能；生成access_token作为非通用的功能，则不需要抽象为接口。
4.“基于接口而非实现编程”这条原则的设计初衷是：分离具体的实现和抽象的接口，封装不稳定的实现，暴露稳定的接口；上游系统不会因为实现的不稳定而变化，基于稳定的接口；更好地实现了解耦，提高了代码的可扩展性。
5.我们在定义接口的时候，一方面，命名要足够通用，不能包含跟具体实现相关的字眼；另一方面，与特定实现有关的方法不要定义在接口中。
总结：我们在做软件开发的时候，一定要有抽象意识、封装意识、接口意识。越抽象、越顶层、越脱离具体某一实现的设计，越能提高代码的灵活性、扩展性、可维护性"
注：实践过程中，采用"简单工厂"进一步优化。
```
